# 8. Checklist final de UX no projeto

Use este checklist para revisar o seu app antes de avançar para as próximas unidades.

## Mobile-first

- [ ] A tela mostra o essencial primeiro
- [ ] A ação principal está em destaque
- [ ] Não há excesso de informação visível

## Navegação

- [ ] O caminho entre telas é curto e previsível
- [ ] Sempre existe uma forma clara de voltar
- [ ] Não há telas “sem saída”

## Feedback visual

- [ ] Botões respondem ao toque
- [ ] Há confirmação de ações importantes
- [ ] O usuário entende quando algo está carregando
- [ ] Existe mensagem para estado vazio
- [ ] Erros são explicados de forma simples
- [ ] O usuário sabe o que fazer em cada estado

## Acessibilidade

- [ ] Texto legível e com bom contraste
- [ ] Elementos tocáveis têm tamanho adequado
- [ ] Formulários têm rótulos claros

## Simplicidade visual

- [ ] A interface tem poucos elementos por tela
- [ ] As ações seguem um padrão visual
- [ ] O layout não distrai o usuário

## Uso com uma mão

- [ ] Ações principais estão na parte inferior
- [ ] Elementos importantes estão ao alcance do polegar
