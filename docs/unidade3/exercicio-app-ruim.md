# 7. Exercício prático: melhorando um app "ruim"

Nesta atividade, você vai receber do professor um app propositalmente mal planejado em termos de UX. O objetivo é **identificar e corrigir problemas de usabilidade**.

## Descrição do projeto

Os projetos de exemplo estão [aqui](../../assets/docs/unit3_projetos_ruins.zip), e foram desenvolvidos de propósito com decisões ruins. São dois projetos: uma que simula um projeto de gorjetas e divisão de contas e outro um gerenciamento de gastos financeiros simples. Eles foram pensados para que você encontre problemas de **estrutura do código** (Unidade 2) e de **experiência do usuário** (Unidade 3).

### O que deve ser melhorado

**1. Estrutura e organização (Unidade 2):**

- Tudo está concentrado em um único componente
- Não há separação entre layout, lista e formulário
- Não existe composable para regras de negócio (tarefas)
- Estilos estão globais e sem organização por partes

**2. Usabilidade e UX (Unidade 3):**

- Layout pensado para desktop, pouco adaptado ao mobile
- Botões pequenos e difíceis de tocar
- Mensagens pouco amigáveis (ex.: alert e textos secos)
- Falta de estados de carregamento/erro/vazio mais claros
- Ações importantes longe do polegar (uso com uma mão)
- Hierarquia visual fraca e excesso de elementos competindo

## Problemas comuns (para observar)

- Botões pequenos ou difíceis de tocar
- Textos confusos
- Falta de feedback visual
- Navegação sem caminho claro
- Excesso de elementos na tela
- Estados de erro e vazio ignorados

## Etapas do exercício

1. Use o app como um usuário comum
2. Liste os principais problemas percebidos
3. Proponha melhorias simples e diretas
4. Reorganize a interface se necessário

## Critérios de avaliação

- Facilidade de uso
- Clareza das ações
- Consistência visual
- Redução de esforço

### Atividade prática

Apresente as melhorias propostas e justifique cada decisão com base nos princípios de UX vistos na unidade.
